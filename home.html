a. What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects; that way, programming languages can interact with the page.

b. How do you select an element from the DOM?

JavaScript provides several methods to select DOM elements. Here are some of them:
Using getElementById:
The getElementById method allows you to select an element by its ID. This method returns the element object if it finds an element with the specified ID, otherwise, it returns null.
let element = document.getElementById("myId");
Using getElementsByClassName:
The getElementsByClassName method allows you to select elements by their class name. It returns a collection of all elements in the document with the specified class name
let elements = document.getElementsByClassName("myClass");
Using getElementsByTagName:
The getElementsByTagName method allows you to select elements by their tag name. It returns a collection of all elements in the document with the specified tag name.
let elements = document.getElementsByTagName("p");
Using querySelector:
The querySelector method allows you to select the first element that matches a specified CSS selector(s) in the document. let element = document.querySelector(".myClass"); Using querySelectorAll: The querySelectorAll method allows you to select all elements that match a specified CSS selector(s) in the document. let elements = document.querySelectorAll(".myClass");

c. What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique where a single event listener is attached to a parent element instead of multiple listeners on child elements. When an event occurs on a child, it bubbles up to the parent, which can handle the event using event.target.
Key benefits;
Efficiency: Reduces the number of event listeners, improving performance.
Dynamic elements: Automatically handles events on newly added child elements.
Cleaner code: Centralizes event handling, making the code easier to maintain.
